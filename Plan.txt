План
- Начальные этапы работы будут проводиться с изображениями, которые удовлетворяют дополнительным требованиям:
    1) не зашумлены лишними предметами
    2) имеют "удобное" освещение (оценивается разработчиком)
    3) на фото обязательно присутствуют и стул и дверь
- Ответ будет только "да" / "нет" (исключается ответ "не знаю")
- Для нахождения контуров проверим различные методы (Canny, Outsu_treshold, ...)
- С помощью применения морфологических функций (opening, closing) можно будет попытать бороться с шумами в контурах
- Также, необходимо, будет подобрать "хорошие" значения параметров для этих методов
- Выделение стула:
    Учитывая, что дверной проём строго прямоугольный, то можно не искать точный контур стула, а ограничить его некоторым четырёхугольником (со сторонами, параллельными сторонам дверного проёма)
 
- Для определения дверного проема:
    можно попытаться искать прямые при помощи метода Хафа
    Возможно, модернизация метода Хафа (выделяет не прямые, а отрезки) позволит откинуть часть прямых, не относящихся к двери (слишком короткие отрезки)
    Но из-за перспективы, нельзя искать прямые под 90 градусов, Также нижний край двери может быть закрыт стулом (А значит дверь может быть в виде буквы "П")
 
- Т.к. сейчас датасет состоит из  картинок конкретной двери и конкретного стула, то, вполне, можно попробовать использовать особые точки для определения гомографии и перспективы.
 Также это позволит сузить область, в которой необходимо искать объекты (дверь, стул)
- Однако, для использования особых точек, придется разметить часть данных
- Если выйдет избавиться от перспективы, то можно будет считать ширину проёма, как расстояние по горизонтали между двумя вертикальными границами двери
- Определение качества результатов, пока, только визуально
